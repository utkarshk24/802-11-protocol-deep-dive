Beamforming: Explicit,
Implicit & Sounding

Beamforming focuses transmit energy toward a specific receiver, raising its SNR without more power. 802.11ac standardized explicit beamforming with a clean sounding protocol — the same machinery that makes downlink MU-MIMO possible. Today we walk the sounding sequence.

⏱~60 min
🎯Beam steering
📻NDP sounding

🎯

Why Beamforming Works

With multiple antennas, an AP can adjust the phase and amplitude of the signal on each so the copies arrive in phase at the target client — constructively adding to a stronger signal — while partially canceling elsewhere. More SNR at the client means a higher sustainable MCS and better range.

  • Beamformer: the transmitter that steers (usually the AP).
  • Beamformee: the receiver being steered toward (the client).
  • Steering matrix (Q): the per-subcarrier weights the beamformer applies.
📻

Explicit vs Implicit

802.11n allowed both; 802.11ac dropped implicit and standardized on explicit:

  • Implicit: The beamformer estimates the channel from frames the client sends, assuming reciprocity. No feedback needed, but calibration is fragile — this is why it was abandoned.
  • Explicit (VHT): The client measures the channel from a sounding frame and sends back compressed feedback. Robust and interoperable, at the cost of some airtime.
🔄

The Sounding Protocol

VHT explicit beamforming uses a Null Data Packet (NDP) sounding sequence:

text NDP Sounding Sequence 
1. AP -> ALL   NDP Announcement (VHT NDPA)
                 "sounding coming; here are the target STAs"
2. AP -> ALL   Null Data Packet (NDP)
                 preamble only (VHT-LTFs), no data payload
3. STA -> AP   VHT Compressed Beamforming feedback
                 the measured steering matrix (Q), compressed as angles
   (for MU: AP polls each additional STA with a
    Beamforming Report Poll before its feedback)
Feedback is compressed to angles.
Sending a full complex channel matrix per subcarrier would be enormous. 802.11ac uses Givens rotations to represent the steering matrix as a set of quantized angles (φ and ψ), shrinking the report dramatically while keeping enough fidelity to steer accurately.
👥

From Beamforming to MU-MIMO

The same steering matrices enable downlink MU-MIMO: the AP computes weights that steer a beam toward client A while placing a null at client B (and vice versa), letting it transmit distinct streams to both at once. Accurate, fresh sounding is essential — stale channel info causes inter-user interference and collapses the gains.
