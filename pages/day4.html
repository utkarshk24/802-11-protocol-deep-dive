MAC Frame Anatomy:
Frame Control & Addresses

Every 802.11 frame starts with a Frame Control field — a 2-byte header that tells you everything: is this a beacon? A data frame? Management? Is it encrypted? Aggregated? Today we decode this header bit-by-bit and understand MAC addressing: source, destination, BSSID, transmitter, receiver.

The 'Why' — Understanding Frame Type at a Glance

One Field Tells You Everything

In Wireshark, you see thousands of frames. How do you instantly know which are beacons, which are data, which are management? The Frame Control field.

Without this knowledge, you see a cryptic hex dump. With it, you can read the frame type, flags, and addressing mode instantly. You'll know if a frame is encrypted (Protected Frame flag), aggregated (A-MSDU), or part of a QoS stream.

Frame Control is the Rosetta Stone of 802.11 protocol debugging.
One 2-byte field tells you frame type, direction, security, QoS, aggregation, and more.
🔍

Frame Control Field: 16 Bits of Information

Bit-by-Bit Breakdown

Frame Control Field (2 bytes = 16 bits):

Bit 0–1:     Protocol Version (2 bits) = 00 (802.11 version 0)
Bit 2–3:     Type (2 bits)              = 00=Mgmt, 01=Control, 10=Data, 11=Ext
Bit 4–7:     Subtype (4 bits)           = Frame subtype within type
Bit 8:       To DS (1 bit)              = 1 if destined for DS (AP→Client path)
Bit 9:       From DS (1 bit)            = 1 if from DS (AP→Client)
Bit 10:      More Frag (1 bit)          = 1 if more fragments coming
Bit 11:      Retry (1 bit)              = 1 if retransmission
Bit 12:      Power Mgmt (1 bit)         = 1 if sender in power-save mode
Bit 13:      More Data (1 bit)          = 1 if more data buffered for receiver
Bit 14:      Protected Frame (1 bit)    = 1 if encrypted (WPA2/WPA3)
Bit 15:      Order (1 bit)              = 1 if HT Order or QoS+A-MPDU or HE Order

Flags Explained

  • To DS / From DS: Indicates AP direction. To DS = client→AP. From DS = AP→client.
  • More Frag: If 1, more fragments follow (receiver must wait for all before reassembly).
  • Retry: If 1, this is a retransmission (same sequence number as before).
  • Power Mgmt: If 1, sender is in power-save sleep (AP should buffer packets for it).
  • More Data: If 1, AP has more packets buffered for this client.
  • Protected Frame: If 1, frame is encrypted (WPA2/WPA3). Critical flag!
  • Order: If 1, uses HT Order (spatial streams) or A-MPDU aggregation or HE Order.

MAC Addressing: RA, TA, DA, SA, BSSID

The Five MAC Addresses (Not Always All Five)

An 802.11 frame can have up to 4 MAC address fields, depending on To DS / From DS flags. Understanding which is which is crucial.

AddressMeaningWhen Present
RAReceiver Address (radio hop destination)Always (first address)
TATransmitter Address (radio hop source)Always (second address)
DADestination Address (final destination)When not AP-to-Client
SASource Address (original source)When not Client-to-AP
BSSIDBasic Service Set ID (AP MAC)Management/Data frames (3rd address)

Examples

Client → AP (To DS = 1, From DS = 0):

RA (Receiver)   = AP MAC address (radio hop destination)
TA (Transmitter) = Client MAC address (radio hop source)
DA (Destination) = Final destination (may be elsewhere on network)
SA (Source)     = Client MAC (original source)
BSSID          = AP MAC address (identifies the network)

AP → Client (To DS = 0, From DS = 1):

RA (Receiver)   = Client MAC address (radio hop destination)
TA (Transmitter) = AP MAC address (radio hop source)
DA (Destination) = Client MAC (final destination)
SA (Source)     = Source on the wired network
BSSID          = AP MAC address

AP → AP (To DS = 1, From DS = 1) — WDS or 4-Address:

RA (Receiver)   = Destination AP MAC
TA (Transmitter) = Source AP MAC
DA (Destination) = Final destination (another AP or network)
SA (Source)     = Original source
BSSID          = (not used; 4th address field contains additional info)
📐

The Complete MAC Header

Frame Control is only the first 2 bytes. Here is the full MAC header of a data frame, field by field. The address fields are variable — a frame carries 3 addresses normally, and a 4th only in the AP-to-AP (WDS/mesh) case:

text 802.11 Data Frame MAC Header 
Bytes  Field
  2    Frame Control
  2    Duration / ID
  6    Address 1  (always present)
  6    Address 2  (usually present)
  6    Address 3  (usually present)
  2    Sequence Control
  6    Address 4  (only if ToDS=1 AND FromDS=1)
  2    QoS Control    (only in QoS Data subtypes)
  4    HT Control     (only if the Order/+HTC bit is set)
 ...   Frame Body     (0 - 2304 bytes, or larger when aggregated)
  4    FCS            (CRC-32 over the whole frame)

Duration / ID — the NAV engine

This 2-byte field usually holds a Duration in microseconds: how long the medium will remain busy for this exchange (this frame + SIFS + ACK, etc.). Every other station hears it and sets its NAV (Network Allocation Vector) — a countdown timer during which it stays silent. This is the "virtual carrier sense" that makes CSMA/CA work without everyone physically hearing everyone. (In PS-Poll frames the same bytes instead carry the station's Association ID.)

Sequence Control — dedup & fragments

12 bits of sequence number (0–4095, incrementing per MSDU) plus 4 bits of fragment number. The receiver uses these to discard duplicate retransmissions and to reassemble fragmented frames. As we saw in the security phase, a sequence number that jumps or regresses for a known transmitter is a spoofing fingerprint.

QoS Control & HT Control

QoS Control (present in QoS Data frames) carries the Traffic Identifier (TID → access category: voice, video, best-effort, background), an EOSP bit, and A-MSDU present bit. HT Control (present when the +HTC/Order bit is set) carries link-adaptation feedback, MIMO power-save, and — in 802.11ax — the buffer-status and OFDMA control fields.

🧭

The Canonical 4-Address Table

This one table is worth memorizing — it tells you exactly what Address 1–4 mean for every combination of the To DS / From DS bits. RA/TA are always the radio-hop receiver and transmitter (Addr1/Addr2); DA/SA are the ultimate endpoints:

ToDSFromDSScenarioAddr1 (RA)Addr2 (TA)Addr3Addr4
00IBSS / control (ad-hoc)DASABSSID—
01AP → client (from DS)DABSSIDSA—
10Client → AP (to DS)BSSIDSADA—
11AP → AP (WDS / mesh)RATADASA
Only the 4th case uses Address 4.
When both DS bits are set, the frame is being bridged between APs, so it needs to carry both the radio hop (RA/TA = Addr1/Addr2) and both true endpoints (DA/SA = Addr3/Addr4). Every other frame gets by with three addresses. This is exactly why the Address 4 field is "only present when ToDS=1 AND FromDS=1."

  • Subtype = QoS Data (VoIP, video streaming, etc.)

Compute It: Decode Frame Control & Addresses

This decodes the 2-byte Frame Control from raw bytes and then, given the DS bits, labels each address using the table above — the exact logic Wireshark applies:

python decode_fc.py 
TYPES = {0: "Management", 1: "Control", 2: "Data", 3: "Extension"}

def decode_frame_control(fc_le):
    """fc_le: the 2 Frame Control bytes as a little-endian int."""
    ver     = fc_le & 0b11
    ftype   = (fc_le >> 2) & 0b11
    subtype = (fc_le >> 4) & 0b1111
    flags = {
        "to_ds":     (fc_le >> 8)  & 1,
        "from_ds":   (fc_le >> 9)  & 1,
        "more_frag": (fc_le >> 10) & 1,
        "retry":     (fc_le >> 11) & 1,
        "pwr_mgmt":  (fc_le >> 12) & 1,
        "more_data": (fc_le >> 13) & 1,
        "protected": (fc_le >> 14) & 1,
        "order":     (fc_le >> 15) & 1,
    }
    return ver, TYPES[ftype], subtype, flags

def label_addresses(to_ds, from_ds):
    return {
        (0, 0): ("DA", "SA", "BSSID", None),
        (0, 1): ("DA", "BSSID", "SA", None),
        (1, 0): ("BSSID", "SA", "DA", None),
        (1, 1): ("RA", "TA", "DA", "SA"),
    }[(to_ds, from_ds)]

# Example: Frame Control 0x8841 (QoS Data, FromDS, Protected)
ver, ftype, sub, flags = decode_frame_control(0x8841)
print(ftype, "subtype", sub, flags)
print("Addr1-4 =", label_addresses(flags["to_ds"], flags["from_ds"]))

🧠 Knowledge Check
What does the Duration/ID field drive?

The NAV - virtual carrier sense - telling other stations how long to stay silent.

When is Address 4 present in a frame?

Only when ToDS=1 AND FromDS=1 (AP-to-AP WDS or mesh bridging).

In a client-to-AP frame (ToDS=1, FromDS=0), which address is the BSSID?

Address 1 (RA) is the BSSID (the AP).
