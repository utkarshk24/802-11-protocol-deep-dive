The VHT Operation IE (ID 192)

Where the VHT Capabilities IE says "what I can do," the VHT Operation IE says "what this BSS is currently doing." Every station reads it from beacons to configure its radio:

  • Channel Width: 0 = 20/40 MHz, 1 = 80 MHz, 2 = 160 MHz, 3 = 80+80 MHz.
  • Channel Center Frequency Segment 0: the center of the (primary) wide channel.
  • Channel Center Frequency Segment 1: the second segment for 160/80+80 MHz.
  • Basic VHT-MCS and NSS Set: the MCS/NSS combinations required to join this BSS.
Capabilities vs Operation — don't confuse them.
Capabilities (191) is a device's menu; Operation (192) is the BSS's current order. A 160 MHz capable AP that is running at 80 MHz advertises 160 in Capabilities but 80 in Operation.
🛡️

Protecting Legacy Devices

802.11a and 802.11n devices can't decode VHT frames, so they can't know how long a VHT transmission will last. Without help they'd transmit on top of it. Several mechanisms prevent this:

  • L-SIG duration spoofing: the legacy L-SIG field (Lesson 20) sets a duration legacy devices honor via their NAV.
  • RTS/CTS & CTS-to-self: the AP reserves the medium with legacy-rate control frames before a VHT burst.
  • Operating Mode Notification: a station can tell the AP it has temporarily narrowed its width or stream count (e.g. to save power), so the AP adapts.

The cost of protection is airtime overhead — mixed legacy/VHT networks run slower than a pure VHT network because every wide transmission needs a legacy-rate guard.

🏢

Channel Coordination Across APs

In an enterprise ESS, a controller (or a distributed algorithm) must assign primary channels so neighboring APs don't put their 80 MHz blocks on top of each other. Best practices:

  • Align primary 20 MHz channels between neighbors so CCA works cleanly (avoid "primary vs secondary" mismatches that cause OBSS problems).
  • Use narrower widths (40 MHz) in high-density deployments to gain more non-overlapping channels.
  • Reserve 160 MHz for isolated APs with clean spectrum.

from scapy.all import rdpcap, Dot11Elt

WIDTH = {0: "20/40 MHz", 1: "80 MHz", 2: "160 MHz", 3: "80+80 MHz"}

for pkt in rdpcap("capture.pcap"):
    elt = pkt.getlayer(Dot11Elt)
    while elt:
        if elt.ID == 192 and len(elt.info) >= 3:   # VHT Operation
            info = bytes(elt.info)
            print(f"Width={WIDTH.get(info[0], info[0])}  "
                  f"seg0_center={info[1]}  seg1_center={info[2]}")
            break
        elt = elt.payload.getlayer(Dot11Elt)
