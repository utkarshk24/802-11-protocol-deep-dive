The Three-State Model

Every station relative to an AP is in exactly one of three states. Frame classes gate what's legal in each state:

  • State 1 — Unauthenticated, Unassociated: Only Class 1 frames allowed (beacons, probes, auth, RTS/CTS/ACK).
  • State 2 — Authenticated, Unassociated: Class 1 + Class 2 (association requests). Reached after successful Auth.
  • State 3 — Authenticated & Associated: All classes, including Class 3 data frames. This is "connected."
text State Transitions 
          Auth success            Assoc success
State 1 -----------------> State 2 -----------------> State 3
   ^                          |                          |
   |     Deauth               |  Deauth                  | Disassoc
   +--------------------------+--------------------------+
   |                                                     |
   +-----------------------------------------------------+
              (Deauth from any state -> State 1)
A Class 3 frame in State 1 is a red flag.
If a station sends data before associating, the AP responds with a deauth (reason 7). Seeing this pattern in a capture often means injection, spoofing, or a confused client — exactly what our analyzer will surface.
🐍

The State Machine Analyzer

This tracker maintains a per-station state and prints every transition as it happens:

python state_analyzer.py 
from scapy.all import sniff, Dot11
from scapy.all import (Dot11Auth, Dot11AssoReq, Dot11AssoResp,
                       Dot11Deauth, Dot11Disas)

# state[mac] = 1 (unauth), 2 (auth), 3 (assoc)
state = {}

def transition(mac, new, why):
    old = state.get(mac, 1)
    if old != new:
        state[mac] = new
        print(f"{mac}:  State {old} -> {new}   ({why})")

def handle(pkt):
    if not pkt.haslayer(Dot11):
        return
    sta = pkt.addr2

    if pkt.haslayer(Dot11Auth) and pkt[Dot11Auth].status == 0:
        transition(sta, 2, "auth success")
    elif pkt.haslayer(Dot11AssoResp) and pkt[Dot11AssoResp].status == 0:
        # response is AP->STA; the STA is addr1
        transition(pkt.addr1, 3, "assoc success")
    elif pkt.haslayer(Dot11Deauth):
        transition(pkt.addr1, 1, "deauth")
        transition(pkt.addr2, 1, "deauth")
    elif pkt.haslayer(Dot11Disas):
        # drop to state 2 (still authenticated)
        for m in (pkt.addr1, pkt.addr2):
            if state.get(m) == 3:
                transition(m, 2, "disassoc")
    elif pkt.type == 2:  # data frame
        if state.get(sta, 1) < 3:
            print(f"⚠️  {sta} sent DATA in state {state.get(sta,1)} "
                  f"(Class 3 violation)")

sniff(iface="wlan0mon", prn=handle, store=0)

Exercise: Extend the Analyzer

  • Task 1: Add timestamps and measure how long each station spends in State 2 (auth-to-assoc latency).
  • Task 2: Track the AID from association responses and print a live table of connected stations.
  • Task 3: Flag any station that transitions to State 1 more than 3 times per minute — a possible deauth-attack victim.
