Power Management:
PS-Poll, TIM/DTIM & U-APSD

The Core Idea: The AP Buffers for Sleepers

A station tells the AP "I'm going to sleep" by setting the Power Management bit (Lesson 4) in a frame's Frame Control. From then on, the AP buffers any frames destined for that station instead of transmitting them. The station's radio powers down (doze state) and wakes only periodically to check whether anything is waiting.

Two states, one bit.
Awake = radio on, can transmit/receive anytime. Doze = radio mostly off, saving power. The Power Management bit is how the station announces transitions. The AP maintains per-station state and a buffer queue keyed by Association ID (Lesson 12).
🗺️

TIM & DTIM: The Beacon's "You've Got Mail" Map

Every beacon (Lesson 9) carries a TIM (Traffic Indication Map) element — a bitmap where each bit corresponds to an Association ID. If the AP has buffered unicast frames for AID 7, it sets bit 7. A dozing station wakes at the beacon, checks its bit, and only stays awake to retrieve data if the bit is set.

Broadcast and multicast frames can't be buffered per-station, so they use the DTIM (Delivery TIM). Every Nth beacon is a DTIM beacon (the DTIM period); the AP sends all buffered group traffic right after it, and all power-save stations must be awake for it:

text TIM / DTIM Timeline (DTIM period = 3) 
Beacon:   B    B    B(DTIM)  B    B    B(DTIM) ...
count:    2    1    0        2    1    0
                    ^ group/multicast delivered here

- Unicast buffered: TIM bit for your AID set in ANY beacon
- Group buffered:   delivered only after a DTIM beacon
- Longer DTIM period = more sleep = more latency for multicast
DTIM period is a battery-vs-latency knob.
DTIM 1 wakes stations every beacon (responsive multicast, worse battery). DTIM 3 lets them sleep through two beacons (better battery, more multicast delay). Misconfigured high DTIM values are a classic cause of laggy multicast/mDNS discovery and choppy wireless casting.
📥

Retrieving Buffered Frames: PS-Poll vs U-APSD

Legacy PS-Poll: when a station sees its TIM bit set, it sends a PS-Poll control frame (which carries its AID in the Duration/ID field). The AP replies with one buffered frame, setting the "More Data" bit if more remain — so the station polls again. Simple, but one round-trip per frame is slow for voice.

U-APSD (Unscheduled Automatic Power Save Delivery / WMM Power Save): the station sends any trigger frame (e.g. an uplink voice packet), and the AP responds by delivering a whole service period of buffered frames at once. Far better for bidirectional real-time traffic like VoIP — one trigger drains the queue.

text Legacy PS-Poll vs U-APSD 
Legacy PS-Poll (one frame per poll):
  STA: wake -> see TIM bit -> PS-Poll -> get 1 frame (More Data=1)
       -> PS-Poll -> get 1 frame ... repeat

U-APSD (service period):
  STA: send trigger (uplink voice) -> AP bursts ALL buffered
       downlink frames -> last frame has EOSP=1 -> STA sleeps

TWT (Lesson 32) is the Wi-Fi 6 evolution: scheduled, negotiated
wake windows instead of "wake every beacon."

Compute It: Read the TIM Element

python tim_parse.py 
from scapy.all import rdpcap, Dot11Beacon, Dot11Elt

def buffered_aids(tim_info):
    """TIM element value: [DTIM count][DTIM period][bmp ctrl][bitmap...]"""
    dtim_count  = tim_info[0]
    dtim_period = tim_info[1]
    bmp_ctrl    = tim_info[2]
    offset      = (bmp_ctrl & 0xFE)      # bitmap offset (even)
    bitmap      = tim_info[3:]
    aids = []
    for byte_i, byte in enumerate(bitmap):
        for bit in range(8):
            if byte & (1 << bit):
                aid = (offset + byte_i) * 8 + bit
                aids.append(aid)
    return dtim_count, dtim_period, aids

for pkt in rdpcap("capture.pcap"):
    if pkt.haslayer(Dot11Beacon):
        elt = pkt.getlayer(Dot11Elt)
        while elt:
            if elt.ID == 5:                    # TIM element
                c, p, aids = buffered_aids(bytes(elt.info))
                print(f"DTIM count={c} period={p}  buffered AIDs={aids}")
                break
            elt = elt.payload.getlayer(Dot11Elt)
        break

🧠 Knowledge Check
How does a station announce it is going to sleep?

It sets the Power Management bit in Frame Control.

TIM vs DTIM?

TIM signals per-AID buffered unicast; DTIM marks when buffered group/multicast is delivered.

PS-Poll vs U-APSD?

PS-Poll fetches one frame per poll; U-APSD delivers a whole service period on a trigger.
