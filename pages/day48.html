Frame Aggregation
& Block Ack

At gigabit PHY rates, the fixed overhead per frame — preamble, IFS, backoff, ACK — dwarfs the payload. Aggregation packs many payloads into one transmission, and Block Ack acknowledges them all at once. Together they're why 802.11n+ throughput is anywhere near its PHY rate.

The 'Why' — Overhead Doesn't Scale With Rate

The PHY got 100× faster from 802.11b to 802.11ac, but SIFS, DIFS, backoff, and the preamble barely changed. Sending a 1500-byte frame at 866 Mbps takes ~14 µs of data — but the surrounding overhead (backoff + preamble + SIFS + ACK) is easily 100+ µs. At high rates you spend most of your airtime not sending data.

The only fix is to send more data per "win."
You already saw TXOP bursting (Lesson 47). Aggregation goes further: combine many payloads into a single PHY frame so the preamble and contention are paid once for kilobytes of data. This is the single biggest MAC-efficiency lever in modern Wi-Fi.
📦

Two Kinds of Aggregation

A-MSDU — aggregate before the MAC header

Multiple MSDUs (the payloads handed down from the network layer) are packed into a single MAC frame with one MAC header and one FCS. Maximally efficient, but fragile: one bit error fails the whole aggregate's FCS and everything must be resent.

A-MPDU — aggregate whole MAC frames

Multiple complete MPDUs (each with its own MAC header and its own FCS) are concatenated behind one PHY preamble, separated by MPDU delimiters. Slightly more overhead, but each subframe is independently checkable — so a single corrupt subframe can be selectively retransmitted while the rest are kept.

text A-MSDU vs A-MPDU 
A-MSDU (one FCS for everything):
  [PHY][MAC hdr][ MSDU1 | MSDU2 | MSDU3 ][FCS]
        one error  ->  entire frame lost

A-MPDU (per-subframe FCS):
  [PHY][ delim|MPDU1|FCS ][ delim|MPDU2|FCS ][ delim|MPDU3|FCS ]
        one error  ->  only that MPDU is retransmitted

Real stacks often nest them: an A-MPDU of subframes,
each subframe itself an A-MSDU. Best of both worlds.

Block Ack: One Ack for Many Frames

ACKing each of 64 subframes individually would destroy the efficiency gain. Block Ack (BA) acknowledges a whole burst with a single frame carrying a bitmap — one bit per subframe, so the sender learns exactly which MPDUs to resend.

text Block Ack Session 
Setup once:   A -> AP  ADDBA Request  (agree buffer size, TID, policy)
              AP -> A  ADDBA Response

Per burst:    A -> AP  A-MPDU (MPDUs with sequence numbers 100..115)
              AP -> A  Block Ack  (bitmap: 1=received, 0=missing)
                        e.g. all 1s except seq 107 -> resend only 107

Teardown:     DELBA when the stream ends
BAR (Block Ack Request) can be sent to solicit a BA if one was lost.
Immediate vs delayed BA.
Almost all modern devices use immediate Block Ack (the BA comes a SIFS after the A-MPDU). The BA's starting sequence number + bitmap is what enables the selective retransmit that makes A-MPDU robust. The receiver keeps a reordering buffer so out-of-order resent frames are delivered to the upper layer in order.
📈

Compute It: Aggregation Efficiency

python aggregation.py 
PREAMBLE = 40      # us, HE-ish preamble
BACKOFF  = 30      # us, avg
SIFS     = 16
ACK      = 32
RATE     = 866e6   # bits/s

def airtime_us(payload_bytes):
    return payload_bytes * 8 / RATE * 1e6

def efficiency(n_frames, payload=1500, aggregated=True):
    data = airtime_us(payload) * n_frames
    if aggregated:                       # one preamble + one Block Ack
        overhead = BACKOFF + PREAMBLE + SIFS + ACK
    else:                                # overhead PER frame
        overhead = (BACKOFF + PREAMBLE + SIFS + ACK) * n_frames
    total = data + overhead
    return data / total * 100

for n in (1, 8, 32, 64):
    agg = efficiency(n, aggregated=True)
    sep = efficiency(n, aggregated=False)
    print(f"{n:3d} frames | aggregated {agg:5.1f}% | separate {sep:5.1f}%")

Run it: with 64 frames, aggregation pushes MAC efficiency from ~30% toward ~90%. That gap is the difference between a PHY rate and real throughput (recall Lesson 27's capstone).
