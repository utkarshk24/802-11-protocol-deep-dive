The Key Hierarchy

Everything descends from the PMK (from SAE, PSK, or 802.1X). The handshake derives the session keys:

text Key Derivation 
PMK  (Pairwise Master Key)   from SAE / PSK / 802.1X
 |
 |  PTK = PRF(PMK, ANonce, SNonce, AP_MAC, STA_MAC)
 v
PTK  (Pairwise Transient Key)  splits into:
       KCK  - Key Confirmation Key (protects handshake MICs)
       KEK  - Key Encryption Key   (encrypts the GTK in msg 3)
       TK   - Temporal Key         (encrypts actual data frames)

GTK  (Group Temporal Key)   shared, for broadcast/multicast frames
                            delivered encrypted under KEK
Nonces make every session key unique.
ANonce (from AP) and SNonce (from STA) are fresh random numbers each connection. Even with the same password, every association produces a different PTK — no key reuse across sessions.
🤝

The 4-Way Handshake

text EAPOL 4-Way Handshake 
Msg 1  AP  -> STA   ANonce
         STA now has ANonce + SNonce -> derives PTK

Msg 2  STA -> AP    SNonce + MIC
         AP now has both nonces -> derives same PTK, verifies MIC

Msg 3  AP  -> STA   GTK (encrypted w/ KEK) + MIC
         STA verifies MIC, installs PTK + GTK

Msg 4  STA -> AP    ACK + MIC
         both install keys; encrypted data can flow

Notice: the PMK is never sent. The nonces and MICs prove both sides know the PMK without revealing it — and confirm they derived identical session keys.

🔄

Group Rekeying & the KRACK Attack

The GTK is shared by all clients, so when a device leaves (or on a timer), the AP rotates it via a 2-way Group Key Handshake to preserve forward/backward secrecy for broadcast traffic.

In 2017, KRACK (Key Reinstallation Attack) showed that an attacker could replay handshake message 3, forcing a client to reinstall an already-in-use PTK and reset its packet-number counter — enabling nonce reuse and, with some ciphers, decryption or forgery. The flaw was in the state machine, not the crypto.

The KRACK fix: never reinstall a key.
Patched clients refuse to reinstall an already-installed PTK/GTK, and treat a retransmitted msg 3 without resetting the nonce counter. KRACK is why keeping supplicant firmware/OS patched matters — the protocol was fine; implementations had to stop being naive.
