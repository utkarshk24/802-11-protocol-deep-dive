Active vs Passive Scanning

In passive scanning a station just listens for beacons on each channel. It's battery-friendly but slow — you wait a full beacon interval per channel.

In active scanning the station transmits a Probe Request (Type 0, Subtype 4) and any AP that matches responds with a Probe Response (Type 0, Subtype 5). This is faster and is how most clients discover networks.

  • Directed probe: SSID IE contains a specific network name.
  • Wildcard/broadcast probe: SSID IE is zero-length — "any AP, respond."
  • Probe Response: Nearly identical body to a beacon (fixed fields + IEs), but unicast to the requesting station.
🕵️

The Privacy Problem & MAC Randomization

Historically, devices sent directed probe requests containing every SSID in their Preferred Network List — "Starbucks", "HomeWiFi", "CorpNet". Anyone sniffing could build a profile of where a person had been. Combined with a fixed MAC address, devices were trivially trackable across malls, airports, and cities.

MAC randomization changed everything.
iOS 8+, Android 9+, and Windows 10+ now transmit probe requests from a randomized, locally-administered MAC (the second-least-significant bit of the first octet is set to 1) and mostly use wildcard SSIDs. This broke naïve device-tracking analytics overnight.

Spotting a Randomized MAC

python mac_check.py 
def is_randomized(mac):
    """Locally-administered bit set => randomized MAC."""
    first_octet = int(mac.split(":")[0], 16)
    return bool(first_octet & 0b10)   # bit 1 (0x02) = local admin

print(is_randomized("aa:bb:cc:dd:ee:ff"))  # True  (0xaa & 2 = 2)
print(is_randomized("00:1a:2b:3c:4d:5e"))  # False (real OUI)

Sniffing Probe Requests

With a card in monitor mode you can watch every probe request in the air. This is the foundation of Wi-Fi presence analytics (and the reason randomization matters):

python probe_sniffer.py 
from scapy.all import sniff, Dot11, Dot11ProbeReq, Dot11Elt

seen = set()

def handle(pkt):
    if pkt.haslayer(Dot11ProbeReq):
        src  = pkt[Dot11].addr2
        ssid = pkt[Dot11Elt].info.decode(errors="ignore") or "<wildcard>"
        key  = (src, ssid)
        if key not in seen:
            seen.add(key)
            local = "randomized" if int(src.split(":")[0], 16) & 2 else "real"
            print(f"{src} ({local}) probing for: {ssid}")

sniff(iface="wlan0mon", prn=handle, store=0)

🧠 Knowledge Check
Directed vs wildcard probe request?

Directed names a specific SSID; wildcard uses a zero-length SSID meaning 'any AP'.

How do you detect a randomized (locally-administered) MAC?

The locally-administered bit (0x02) is set in the first octet.

Why did MAC randomization matter?

It broke passive device tracking that relied on probe SSIDs plus a fixed MAC.
