Every connection ends. Deauthentication and disassociation frames tear down associations — and because legacy management frames are unauthenticated, they became the most famous attack vector in Wi-Fi. Today: reason codes, the deauth flood, and the MFP defense.

Two Ways to Say Goodbye

  • Deauthentication (Subtype 12): Terminates authentication. Since association requires authentication, deauth implicitly ends the association too. It's the "harder" reset.
  • Disassociation (Subtype 10): Ends only the association but keeps authentication. Often used when an AP is shedding load or a client is roaming.

Both carry a single fixed field: a 2-byte Reason Code.

text Common Reason Codes 
 1  Unspecified reason
 2  Previous auth no longer valid
 3  Deauth because sending station is leaving
 4  Disassoc due to inactivity
 5  AP unable to handle all associated stations
 6  Class 2 frame from non-authenticated station
 7  Class 3 frame from non-associated station
 8  Disassoc because station is leaving
15  4-way handshake timeout
16  Group key handshake timeout
💥

The Deauth Flood Attack

Because pre-802.11w management frames have no cryptographic protection, an attacker can spoof a deauth frame pretending to be the AP. The victim's station believes the AP kicked it and disconnects. Send these continuously and the victim can never stay online.

⚠️ Educational context only.
Understanding this attack is essential for defense and authorized testing. Sending deauth frames against networks you don't own is illegal in most jurisdictions. The detection code below is the defensive takeaway.

This underpins evil-twin attacks (force a client off the real AP so it reconnects to a rogue one) and captive-portal harvesting.

🛡️

Detecting Deauth Floods & the MFP Fix

A defensive monitor watches for an abnormal rate of deauth/disassoc frames from any BSSID:

python deauth_detector.py 
from scapy.all import sniff, Dot11, Dot11Deauth, Dot11Disas
from collections import defaultdict
import time

counts = defaultdict(list)
WINDOW = 10      # seconds
THRESHOLD = 20   # frames per window = suspicious

def handle(pkt):
    if pkt.haslayer(Dot11Deauth) or pkt.haslayer(Dot11Disas):
        now = time.time()
        src = pkt[Dot11].addr2
        counts[src] = [t for t in counts[src] if now - t < WINDOW]
        counts[src].append(now)
        if len(counts[src]) > THRESHOLD:
            print(f"⚠️  Possible deauth flood from {src}: "
                  f"{len(counts[src])} frames / {WINDOW}s")

sniff(iface="wlan0mon", prn=handle, store=0)

The real fix is 802.11w Management Frame Protection (MFP), which adds a MIC to deauth/disassoc frames so they can't be spoofed. WPA3 makes MFP mandatory — we cover it in depth on Lesson 42.
