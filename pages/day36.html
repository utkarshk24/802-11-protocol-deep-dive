The Experiment

Model N clients each with a small payload. In OFDM, each transmission pays the full overhead (DIFS + backoff + preamble + SIFS + ACK) and they go one at a time. In OFDMA, up to one PPDU's worth of clients share the overhead once. We compute total airtime for both.

Overhead is fixed per transmission — so amortize it.
For tiny packets the overhead dwarfs the payload time. OFDMA's whole value is paying that fixed cost once for many clients instead of once per client. The simulator makes the multiplier obvious.
🐍

The Efficiency Simulator

python ofdma_sim.py 
# Simplified airtime model (microseconds)
DIFS      = 34
SIFS      = 16
BACKOFF   = 67.5     # avg for AC_BE (CW=15 slots * 9us / 2)
PREAMBLE  = 40       # HE preamble-ish
ACK       = 32
RATE_MBPS = 100      # per-client effective rate on its RU/channel

def tx_time_us(payload_bytes, rate_mbps=RATE_MBPS):
    return payload_bytes * 8 / rate_mbps    # us

def ofdm_airtime(n_clients, payload):
    """Each client contends and transmits separately."""
    per = DIFS + BACKOFF + PREAMBLE + tx_time_us(payload) + SIFS + ACK
    return n_clients * per

def ofdma_airtime(n_clients, payload, ru_per_ppdu=9):
    """Clients packed into PPDUs of up to ru_per_ppdu each."""
    ppdus = -(-n_clients // ru_per_ppdu)     # ceil division
    # one PPDU pays overhead once; payload time is the max RU's time
    per_ppdu = DIFS + BACKOFF + PREAMBLE + tx_time_us(payload) + SIFS + ACK
    return ppdus * per_ppdu

for n in (1, 9, 30, 100):
    payload = 200                            # small IoT packet
    a = ofdm_airtime(n, payload)
    b = ofdma_airtime(n, payload)
    print(f"{n:3d} clients | OFDM {a:9.0f} us | OFDMA {b:8.0f} us | "
          f"{a / b:4.1f}x faster")

Run it: with 30 small-packet clients you'll see OFDMA finish several times faster — exactly the "up to 4× efficiency in dense networks" claim, reproduced from first principles.
