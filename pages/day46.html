Channel Access:
CSMA/CA, DCF & the NAV

Everything in Phases 3–4 — OFDMA, MU-EDCA, TXOPs — sits on top of one foundational mechanism: how a station decides when it's allowed to transmit on a shared, half-duplex medium. This is the MAC's beating heart, and it was never given a dedicated lesson. Today we fix that.

The 'Why' — You Can't Detect Collisions in the Air

Wired Ethernet uses CSMA/CD (Collision Detection): a station transmits and listens simultaneously, and if it hears a collision it stops. Radio can't do that — a Wi-Fi radio is half-duplex and its own transmission is millions of times louder than any incoming signal, so it's deaf while talking. It literally cannot hear a collision happen.

So 802.11 uses CSMA/CA (Collision Avoidance): stations work hard to avoid transmitting at the same time in the first place, and confirm success with an explicit ACK. No ACK = assume collision = retransmit.

Every frame is acknowledged (almost).
Because collisions can't be detected, the receiver sends an ACK after every successfully received unicast frame (a SIFS later). Missing ACK is the only collision signal a sender gets. Broadcast/multicast frames are not ACKed — which is why they're sent at low, robust rates.
⏱️

DCF: Listen, Wait, Backoff, Transmit

The Distributed Coordination Function (DCF) is the base access method. Before transmitting, a station must sense the medium idle for a fixed interframe space, then wait a random extra backoff. The interframe spaces create a priority hierarchy — shorter wait = higher priority:

text Interframe Spaces (IFS) 
Name   Purpose                        5 GHz     2.4 GHz (HR/DSSS)
SIFS   before ACK/CTS/response        16 us     10 us
slot   backoff slot time              9 us      20 us (9 us if short slot)
PIFS   PCF / point coordinator        SIFS+1slot
DIFS   normal DCF access              SIFS+2slot = 34 us   50 us
EIFS   after a bad/errored frame      SIFS + ACKtime + DIFS  (long)

Shorter IFS = higher priority. SIFS is shortest, so ACKs and CTS
always win the medium before any new DIFS-based transmission starts.

Random Backoff & the Contention Window

If two stations both waited exactly DIFS they'd collide every time. So after DIFS, each picks a random backoff = random(0, CW) × slotTime and counts down, but only while the medium stays idle (the counter freezes when someone else transmits). The station whose counter hits zero first transmits. On a collision (no ACK), the Contention Window doubles — binary exponential backoff:

text Binary Exponential Backoff 
CWmin = 15, CWmax = 1023  (typical AC_BE, in slots)

attempt 1: CW = 15    -> pick random(0..15)
attempt 2: CW = 31    -> pick random(0..31)   (doubled after failure)
attempt 3: CW = 63
...
capped at CWmax = 1023
success  -> reset CW back to CWmin
🛡️

Virtual Carrier Sense: the NAV & RTS/CTS

Physical carrier sense isn't enough because of the hidden node problem: two stations on opposite sides of an AP can't hear each other, so both think the medium is idle and collide at the AP. The fix is virtual carrier sense using the NAV (Network Allocation Vector).

Every frame's Duration/ID field (Lesson 4) says how long the rest of the exchange will take. Stations that hear it set their NAV — a countdown during which they treat the medium as busy even if they hear nothing. The optional RTS/CTS handshake reserves the medium explicitly:

text RTS/CTS Exchange 
A -> AP   RTS   (Duration covers CTS+data+ACK)   everyone near A sets NAV
AP -> A   CTS   (Duration covers data+ACK)        everyone near AP sets NAV
A -> AP   DATA
AP -> A   ACK

Now even a hidden node C (hears AP but not A) saw the CTS,
set its NAV, and stays silent -> no collision at the AP.
RTS/CTS costs airtime — it's a threshold.
The two extra control frames are pure overhead, so RTS/CTS only kicks in above a configurable RTS threshold (frame size). Small frames skip it; large frames (where a collision wastes more airtime) use it. In dense/hidden-node environments, lowering the threshold trades raw throughput for fewer collisions.
🐍

Compute It: A Backoff Simulator

This models two stations contending under DCF so you can watch collisions trigger CW doubling and see why fairness emerges statistically:

python dcf_backoff.py 
import random

def contend(n_stations=2, rounds=10, cw_min=15, cw_max=1023):
    cw = [cw_min] * n_stations
    backoff = [random.randint(0, cw[i]) for i in range(n_stations)]
    wins = [0] * n_stations

    for r in range(rounds):
        low = min(backoff)
        winners = [i for i, b in enumerate(backoff) if b == low]

        if len(winners) == 1:                 # clean transmission
            w = winners[0]
            wins[w] += 1
            cw[w] = cw_min                     # reset on success
            backoff[w] = random.randint(0, cw[w])
            for i in range(n_stations):        # others decrement
                if i != w:
                    backoff[i] -= low
        else:                                  # collision: >1 hit 0
            for i in winners:
                cw[i] = min(cw[i] * 2 + 1, cw_max)   # double CW
                backoff[i] = random.randint(0, cw[i])
            print(f"round {r}: COLLISION among {winners}, "
                  f"new CW={[cw[i] for i in winners]}")
    return wins

print("wins per station:", contend())

🧠 Knowledge Check
Why CSMA/CA instead of CSMA/CD?

Radios are half-duplex and deaf while transmitting, so they cannot detect collisions.

What is DIFS on 5 GHz?

SIFS (16) + 2 x slot (9) = 34 microseconds.

What solves the hidden-node problem?

Virtual carrier sense (the NAV), optionally reinforced by RTS/CTS.
