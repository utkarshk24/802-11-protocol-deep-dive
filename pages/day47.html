QoS & EDCA:
Access Categories, TXOP & WMM

Plain DCF (Lesson 46) is fair but blind — a bulk file download contends on equal terms with a VoIP call, and the call suffers. 802.11e introduced EDCA to give latency-sensitive traffic priority. It's the foundation MU-EDCA (Lesson 33) extended for Wi-Fi 6.

Four Access Categories

EDCA (Enhanced Distributed Channel Access) runs four independent DCF-like backoff instances inside every station, one per Access Category (AC). Each AC has its own AIFS (arbitration IFS), CWmin/CWmax, and TXOP limit. Higher-priority ACs wait less and back off less, so they statistically win the medium:

Access CategoryTrafficAIFSNCWminCWmaxTXOP limit
AC_VO (Voice)VoIP2371.504 ms
AC_VI (Video)streaming27153.008 ms
AC_BE (Best Effort)web, email31510230 (single frame)
AC_BK (Background)backups, P2P71510230
Priority = smaller numbers.
Voice uses AIFSN 2 and CWmin 3, so it waits ~34 µs and picks a backoff from just 0–3 slots. Background uses AIFSN 7 and CWmin 15 — much longer waits. Over many transmissions, voice consistently jumps the queue. AIFS = SIFS + AIFSN × slot.
🏷️

How Traffic Gets Classified

A frame lands in an AC based on its TID (Traffic Identifier) in the QoS Control field (Lesson 4), which maps from the 802.1D user priority (often derived from the IP DSCP value). The mapping:

text 802.1D UP / TID -> Access Category 
UP/TID  802.1D name        Access Category
 1, 2   Background          AC_BK
 0, 3   Best Effort         AC_BE
 4, 5   Video               AC_VI
 6, 7   Voice               AC_VO

Common DSCP mapping:  EF (46) -> Voice,  AF41 (34) -> Video,
                      default (0) -> Best Effort
Watch the DSCP-to-UP trust boundary.
A malicious or misconfigured client can mark all its traffic EF (voice) to grab priority. Good network design re-marks or polices DSCP at the edge rather than blindly trusting client markings — otherwise EDCA priority becomes a free-for-all.
⚡

TXOP: Winning the Medium, Then Bursting

Contending for every single frame is wasteful. A Transmit Opportunity (TXOP) lets a station that wins the medium hold it for a bounded time and send multiple frames back-to-back, separated only by SIFS (no re-contention). Voice gets a short TXOP; video gets a longer one so it can push several packets or an aggregate:

text TXOP Burst 
win medium (AIFS + backoff)
  |-- DATA1 -- SIFS -- ACK -- SIFS -- DATA2 -- SIFS -- ACK --| ... until TXOP limit
  (no new backoff between frames; the station "owns" the channel)

TXOP=0 means "one frame per win" (best-effort/background).
Larger TXOP amortizes contention overhead across many frames.

The Wi-Fi Alliance certifies EDCA under the name WMM (Wi-Fi Multimedia) — when a device says "WMM certified," it implements these four ACs. WMM is a prerequisite for 802.11n/ac/ax high throughput: without QoS enabled, HT/VHT/HE rates aren't even allowed.

🐍

Compute It: Classify & Compare AIFS

python edca.py 
SIFS, SLOT = 16, 9   # microseconds, 5 GHz

# AC: (AIFSN, CWmin, CWmax, TXOP_us)
AC = {
    "AC_VO": (2, 3, 7, 1504),
    "AC_VI": (2, 7, 15, 3008),
    "AC_BE": (3, 15, 1023, 0),
    "AC_BK": (7, 15, 1023, 0),
}

def aifs(aifsn):
    return SIFS + aifsn * SLOT

def tid_to_ac(tid):
    return {1: "AC_BK", 2: "AC_BK", 0: "AC_BE", 3: "AC_BE",
            4: "AC_VI", 5: "AC_VI", 6: "AC_VO", 7: "AC_VO"}[tid]

for name, (aifsn, cwmin, cwmax, txop) in AC.items():
    print(f"{name}: AIFS={aifs(aifsn):3d}us  CWmin={cwmin:4d}  TXOP={txop}us")

print("\nTID 6 (voice) ->", tid_to_ac(6))
print("TID 0 (web)   ->", tid_to_ac(0))

🧠 Knowledge Check
What are the four EDCA access categories?

Voice, Video, Best Effort, and Background.

Why does voice win the medium more often?

It uses a smaller AIFSN and CWmin, so it waits and backs off less.

What is a TXOP?

A bounded time to burst multiple frames back-to-back without re-contending.
