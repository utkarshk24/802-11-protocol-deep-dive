The Action Frame Model

Every Action frame body starts with a Category byte, then an Action byte, then category-specific fields. This TLV-ish design let the IEEE add features across amendments without new frame subtypes.

text Key Action Categories 
 0  Spectrum Management   TPC, Channel Switch Announcement (802.11h)
 1  QoS                   ADDTS / DELTS traffic streams
 3  Block Ack             ADDBA / DELBA aggregation setup
 5  Radio Measurement     802.11k neighbor reports, beacon requests
 6  Fast BSS Transition   802.11r FT action
 7  HT                    802.11n notify channel width, SM power save
10  WNM                   802.11v BSS Transition Management
21  VHT                   802.11ac group ID, operating mode notify
126 Vendor Specific       vendor extensions

Channel Switch Announcement (CSA)

When an AP must move channels — typically because DFS radar was detected on a 5 GHz channel — it can't just vanish. It broadcasts a Channel Switch Announcement (in beacons and as a Spectrum Management action frame) telling every associated station: "switch to channel X in N beacon intervals."

  • Channel Switch Mode: 1 = stop transmitting until the switch (radar safety).
  • New Channel Number: where everyone is moving.
  • Channel Switch Count: beacons remaining before the switch happens — lets stations follow without disconnecting.
DFS makes CSA mandatory.
On DFS channels (many of the 5 GHz U-NII-2 range), APs must vacate within seconds of detecting radar. CSA is what makes that graceful instead of dropping every client. If you've seen a Wi-Fi network briefly stutter and continue, you likely witnessed a DFS-triggered CSA.
📊

802.11k Radio Measurement

Radio Measurement action frames let an AP ask a station to report on its RF environment — neighbor APs, channel load, noise histograms. The Neighbor Report is the big one: it hands the client a curated list of roam candidates so it doesn't have to scan every channel blindly, dramatically speeding up roaming decisions.

python action_classifier.py 
from scapy.all import rdpcap, Dot11

CATEGORY = {0: "Spectrum Mgmt", 1: "QoS", 3: "Block Ack",
            5: "Radio Measurement", 6: "FT", 10: "WNM", 21: "VHT"}

for pkt in rdpcap("capture.pcap"):
    # Action frames: type=0 (mgmt), subtype=13
    if pkt.haslayer(Dot11) and pkt.type == 0 and pkt.subtype == 13:
        body = bytes(pkt[Dot11].payload)
        if body:
            cat = body[0]
            print(f"{pkt.addr2} -> {pkt.addr1}  "
                  f"Action category={CATEGORY.get(cat, cat)}")
