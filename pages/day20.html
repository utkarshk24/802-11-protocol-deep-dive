& PLCP Headers

Before a receiver can decode the MAC frame, it must parse the PHY preamble. The VHT PPDU carries training fields and two signal fields — VHT-SIG-A and VHT-SIG-B — that tell the radio the bandwidth, MCS, streams, and coding. Today we walk the whole preamble.

The VHT PPDU Layout

A PPDU (PLCP Protocol Data Unit) is the complete over-the-air frame. The VHT format begins with legacy fields so older 802.11a/n devices can at least detect the frame and defer:

text VHT PPDU Fields (in order) 
L-STF     Legacy Short Training Field   AGC, timing sync
L-LTF     Legacy Long Training Field    channel estimation (legacy)
L-SIG     Legacy Signal Field           rate+length so legacy defers
VHT-SIG-A Signal A (2 symbols)          BW, STBC, Group ID, NSS, coding, MCS...
VHT-STF   Short Training Field          AGC for MIMO
VHT-LTF   Long Training Fields (1/NSS)  MIMO channel estimation
VHT-SIG-B Signal B                      per-user length + MCS (esp. MU-MIMO)
Data      PSDU                          the actual MAC frame(s)
Why the legacy preamble?
L-STF/L-LTF/L-SIG let 802.11a/n devices — which can't decode VHT — still read the L-SIG duration and set their NAV, avoiding collisions. This "spoofing" of a legacy header is how 802.11ac stays a good neighbor in mixed networks.
📻

VHT-SIG-A: The Rate Descriptor

VHT-SIG-A is two OFDM symbols carrying the parameters the receiver needs to configure its demodulator:

  • BW (2 bits): 0=20, 1=40, 2=80, 3=160/80+80 MHz.
  • STBC (1 bit): space-time block coding on/off.
  • Group ID (6 bits): 0 or 63 = single-user (SU); 1–62 = MU-MIMO group.
  • NSTS / Partial AID: number of space-time streams (SU) or per-user stream allocation (MU).
  • Short GI (1 bit): 800 ns vs 400 ns guard interval.
  • Coding + MCS (SU): BCC vs LDPC, and the modulation-and-coding-scheme index 0–9.
🧩

VHT-SIG-B: Per-User Detail

In single-user frames, VHT-SIG-B mainly carries the payload length. Its real power shows in MU-MIMO: it delivers each user's individual length and MCS so several clients can be served in one PPDU. This is the field that makes multi-user downlink work.

With radiotap headers on a monitor-mode capture, most of these VHT parameters are surfaced so you can read them without demodulating the RF yourself:

python vht_radiotap.py 
from scapy.all import rdpcap, RadioTap

for pkt in rdpcap("vht_capture.pcap"):
    if pkt.haslayer(RadioTap):
        rt = pkt[RadioTap]
        # field names vary by scapy version / driver
        for f in ("ChannelFrequency", "MCS_index", "bandwidth",
                  "dBm_AntSignal"):
            if hasattr(rt, f):
                print(f"{f:18s} = {getattr(rt, f)}")
        print("-" * 30)

🧠 Knowledge Check
Why keep the legacy preamble in a VHT frame?

So 802.11a/n devices detect the frame and set their NAV (coexistence).

What does VHT-SIG-A carry?

Bandwidth, STBC, Group ID, streams, coding, and MCS.

Why is VHT-SIG-B important for MU-MIMO?

It carries each user's length and MCS so multiple clients decode their own part.
