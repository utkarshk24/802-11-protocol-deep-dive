Beacon Frame Structure

The Fixed Fields

A beacon is a management frame (Type 0, Subtype 8). After the MAC header, the frame body starts with three fixed fields that appear in every beacon:

  • Timestamp (8 bytes): The AP's TSF timer in microseconds. Stations sync their clocks to this — critical for power save and channel hopping.
  • Beacon Interval (2 bytes): Time between beacons in Time Units (1 TU = 1024 µs). Default 100 TU = 102.4 ms.
  • Capability Info (2 bytes): Bitmask flags — ESS/IBSS, Privacy (WEP-era flag), Short Preamble, Spectrum Management, QoS.

Information Elements (IEs)

After the fixed fields come the tagged parameters — Information Elements. Each IE is TLV-encoded: Element ID (1 byte) + Length (1 byte) + Value (N bytes).

text Common Beacon IEs 
ID   0  SSID                  Network name (0-32 bytes)
ID   1  Supported Rates       Basic + supported rates
ID   3  DS Parameter Set      Current channel
ID   5  TIM                   Traffic Indication Map (power save)
ID  32  Power Constraint      802.11h TPC
ID  45  HT Capabilities       802.11n
ID  48  RSN                   WPA2/WPA3 security suite
ID  61  HT Operation          Channel width, secondary channel
ID 191  VHT Capabilities      802.11ac
ID 192  VHT Operation         80/160 MHz channel config
ID 221  Vendor Specific       WPA1, WMM, WPS, vendor extensions
ID 255  Extension (HE/EHT)    802.11ax/be elements use EID extension

Parsing Beacons with Scapy

Scapy exposes IEs as a linked list of Dot11Elt layers. Walk the chain to extract every element:

python beacon_parser.py 
from scapy.all import rdpcap, Dot11, Dot11Beacon, Dot11Elt

packets = rdpcap("capture.pcap")

for pkt in packets:
    if pkt.haslayer(Dot11Beacon):
        bssid = pkt[Dot11].addr3
        stats = pkt[Dot11Beacon].network_stats()

        print(f"BSSID:    {bssid}")
        print(f"SSID:     {stats.get('ssid')}")
        print(f"Channel:  {stats.get('channel')}")
        print(f"Rates:    {stats.get('rates')}")
        print(f"Crypto:   {stats.get('crypto')}")

        # Walk every Information Element manually
        elt = pkt.getlayer(Dot11Elt)
        while elt:
            print(f"  IE {elt.ID:3d}  len={elt.len:3d}  {elt.info[:24]}")
            elt = elt.payload.getlayer(Dot11Elt)
        break

Beacon Interval Analysis

Measuring actual inter-beacon gaps reveals AP health. Jitter above a few milliseconds signals congestion or an overloaded AP radio:

python beacon_timing.py 
times = [float(p.time) for p in packets
         if p.haslayer(Dot11Beacon) and p[Dot11].addr3 == "aa:bb:cc:dd:ee:ff"]

gaps = [(t2 - t1) * 1000 for t1, t2 in zip(times, times[1:])]
avg  = sum(gaps) / len(gaps)

print(f"Beacons captured : {len(times)}")
print(f"Average interval : {avg:.1f} ms (expected ~102.4)")
print(f"Max jitter       : {max(abs(g - avg) for g in gaps):.1f} ms")

🧠 Knowledge Check
What three fixed fields begin a beacon body?

Timestamp, Beacon Interval, and Capability Info.

How are Information Elements encoded?

As TLV: Element ID, Length, then Value.

What is a typical beacon interval?

100 TU, about 102.4 ms (1 TU = 1024 microseconds).
