The 'Why' — From Frequency to Speed

Frequency Gives You Bandwidth. Modulation Gives You Speed.

A 20 MHz 2.4 GHz channel has a fixed bandwidth. You can't make it wider. But you CAN change how much data you squeeze into that bandwidth by using smarter modulation.

BPSK: 1 bit per symbol → 6 Mbps on a 20 MHz channel QPSK: 2 bits per symbol → 12 Mbps on the same channel 16-QAM: 4 bits per symbol → 24 Mbps 64-QAM: 6 bits per symbol → 36 Mbps

Same frequency. Same channel width. Different modulation = different speed. But there's a tradeoff: higher modulation requires cleaner signal. At -85 dBm, 64-QAM fails. You fall back to BPSK.

Modulation is the engineer's speed vs. range dial.
Want maximum range? Use BPSK. Want maximum speed? Use 64-QAM. OFDM (802.11a/g/n/ac/ax) uses both simultaneously: subcarriers near the AP use 64-QAM, farther clients use BPSK. You get high average throughput across the whole network.

Modulation Schemes: From BPSK to 1024-QAM

How Modulation Works

Radio waves are sinusoids. Modulation changes three properties:

  • Amplitude: How loud the wave is (rarely used in 802.11; too vulnerable to interference)
  • Frequency: How fast it oscillates (rarely used alone; OFDM uses many subcarriers)
  • Phase: Where in the cycle it starts (this is the key for 802.11)

Phase Shift Keying (PSK): The 802.11 Workhorse

PSK modulation encodes data in the phase of the carrier wave. Different phase angles represent different bit patterns.

ModulationPhase StatesBits/SymbolMin RSSIUse Case
BPSK2 (0°, 180°)1-82 dBmLongest range, slowest
QPSK4 (0°, 90°, 180°, 270°)2-79 dBmBalanced
16-QAM16 states4-74 dBmGood signal needed
64-QAM64 states6-65 dBmExcellent signal only
256-QAM256 states8-55 dBmVery close range

Why the tradeoff? When you have 64 different phase angles, they're spaced only 5.6° apart. Noise or interference can cause the receiver to misinterpret a signal and read the wrong phase → bit error. BPSK has only 180° separation, so it's very robust.

OFDM: The 802.11a/g Breakthrough

Single Carrier Problem

802.11b uses a single carrier modulated with BPSK/QPSK. It's simple but slow: max 11 Mbps. The problem: if you try to send data faster, the signal becomes more fragile in multipath environments (reflections from walls, furniture).

OFDM Solution: Parallel Subcarriers

OFDM (Orthogonal Frequency Division Multiplexing) splits the channel into many narrowband subcarriers, each sending a piece of the data in parallel. 

802.11b (single carrier):
  20 MHz channel → 1 carrier → 11 Mbps max

802.11a/g (OFDM):
  20 MHz channel → 52 subcarriers (48 data + 4 pilot) → 54 Mbps max

802.11n (wider OFDM):
  40 MHz channel → 108 subcarriers → 150 Mbps (plus spatial streams)

802.11ac (wide + more spatial streams):
  80 MHz channel → 234 subcarriers → 433 Mbps (1x spatial stream)
  80 MHz channel → 234 subcarriers → 866 Mbps (2x spatial streams)

Why Subcarriers Work

By splitting the signal into many narrowband pieces, OFDM makes each subcarrier robust to multipath (reflections don't hurt narrow signals as much). You can use higher modulation (64-QAM, 256-QAM) on each subcarrier without the fragility.

Subcarrier Spacing: 312.5 kHz (802.11a/g/n/ac/ax)

20 MHz channel:
  Bandwidth ÷ subcarrier spacing = number of subcarriers
  20,000 kHz ÷ 312.5 kHz = 64 subcarriers
  (52 data + 4 pilot + 8 null/guard band)

40 MHz channel:
  40,000 kHz ÷ 312.5 kHz = 128 subcarriers

80 MHz channel:
  80,000 kHz ÷ 312.5 kHz = 256 subcarriers

160 MHz channel:
  160,000 kHz ÷ 312.5 kHz = 512 subcarriers

Calculating Data Rates from MCS & Channel Width

MCS: Modulation & Coding Scheme

MCS (0–11 for 802.11a/g/n) encodes both modulation and coding rate:

MCSModulationCoding RateBits/Symbol20 MHz40 MHz
0BPSK1/20.56 Mbps13.5 Mbps
1QPSK1/2112 Mbps27 Mbps
2QPSK3/41.518 Mbps40.5 Mbps
316-QAM1/2224 Mbps54 Mbps
416-QAM3/4336 Mbps81 Mbps
564-QAM2/3448 Mbps108 Mbps
664-QAM3/44.554 Mbps121.5 Mbps
764-QAM5/6560 Mbps (802.11g)135 Mbps

The Formula

Data Rate = (bits per symbol) × (number of subcarriers) × (symbol rate)

For 802.11a/g/n:
  Symbol rate = 250,000 symbols/sec (fixed)

Example: 20 MHz, MCS 5 (64-QAM, 2/3 rate):
  Bits per symbol = 4
  Subcarriers = 52 (data only, not pilot)
  Data Rate = 4 × 52 × 250,000 / 1,000,000 = 48 Mbps

Example: 40 MHz, MCS 5:
  Subcarriers = 108
  Data Rate = 4 × 108 × 250,000 / 1,000,000 = 108 Mbps

Coding Rate: Why 1/2, 3/4, 5/6?

Coding rate is a ratio of useful data to total transmitted bits.

  • 1/2 rate: 1 useful bit per 2 transmitted bits. Half is error-correction overhead. Very robust.
  • 3/4 rate: 3 useful bits per 4 transmitted bits. Less overhead, less robust than 1/2.
  • 5/6 rate: 5 useful bits per 6 transmitted bits. Minimal overhead, fragile in noise.

The FEC (Forward Error Correction) algorithm uses extra bits to reconstruct corrupted data. Lower coding rates = more redundancy = more robust in bad signal.

802.11 MAC Header
  Frame Control: 0x0841
  Flags: ToDS
  Duration/ID: 0 (QoS)
  Receiver Address: aa:bb:cc:dd:ee:01
  Transmitter Address: aa:bb:cc:dd:ee:02

What this tells you: The frame was sent at 54 Mbps (MCS 6) with excellent signal (-45 dBm). This is near the AP. If the same device was 100 meters away, you'd see a lower data rate like 12 Mbps (MCS 1) because signal would be weaker.

Compute It: An OFDM Data-Rate Calculator

This reproduces the standard MCS rate tables from first principles, so you can verify any published number and see exactly how bandwidth, modulation, coding, and guard interval combine.

python ofdm_rate.py 
# data subcarriers per channel width (802.11n/ac numbering)
DATA_SC = {20: 52, 40: 108, 80: 234, 160: 468}
# bits per subcarrier by modulation
BPSCS = {"BPSK": 1, "QPSK": 2, "16-QAM": 4, "64-QAM": 6,
         "256-QAM": 8, "1024-QAM": 10}

def rate_mbps(modulation, coding, bw_mhz, nss=1, short_gi=False):
    sc   = DATA_SC[bw_mhz]
    bits = BPSCS[modulation] * coding * sc * nss
    t_sym = 3.2e-6 + (0.4e-6 if short_gi else 0.8e-6)   # symbol + GI
    return bits / t_sym / 1e6

# 802.11n MCS 7: 64-QAM, 5/6 coding, 20 MHz, 1 stream, long GI
print(f"MCS7  20MHz 1SS  = {rate_mbps('64-QAM', 5/6, 20):.1f} Mbps")   # ~65
# 802.11ac MCS 9: 256-QAM 5/6, 80 MHz, 2 streams, short GI
print(f"VHT9  80MHz 2SS  = {rate_mbps('256-QAM', 5/6, 80, 2, True):.1f} Mbps")  # ~867
# 802.11ax 1024-QAM, 160 MHz, 2 streams, short GI
print(f"HE11 160MHz 2SS  = {rate_mbps('1024-QAM', 5/6, 160, 2, True):.1f} Mbps")
Two ways to reach the same number.
Earlier we used bits/symbol × subcarriers × 250,000 symbols/s; here we divide by the symbol time (3.2 µs + guard interval). They're identical — 1 ÷ 4 µs = 250,000. Learn the time-based form; it generalizes cleanly to short guard intervals and 802.11ax's longer symbols.
